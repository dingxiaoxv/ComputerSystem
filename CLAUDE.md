```

gdb                   查看栈、寄存器、内存
readelf               查看 ELF、节区、符号
ldd                   查看动态库依赖
```

## 验收标准

学习结束后能清楚回答：

1. 一个 C 程序从源码到运行，经历了哪些阶段
2. 为什么某些整数比较会出错
3. 一个函数调用时，栈和寄存器发生了什么
4. 为什么程序会出现段错误
5. 为什么两段逻辑相同的代码性能差很多（要能从 cache miss 角度解释）
6. `perf stat` 输出里每个数字代表什么，怎么判断程序瓶颈在哪
7. 为什么多线程程序会出现竞态和死锁，以及 false sharing 是什么
