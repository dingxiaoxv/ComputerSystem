# §2.2 整数表示（无符号数与补码）总结

## 两种编码的本质差异

同样的 4 位二进制 `1000`，两种编码给出不同的数值：

```
位模式:   1    0    0    0
         ↓    ↓    ↓    ↓
权重:    +8   +4   +2   +1   → 无符号: 8
权重:    -8   +4   +2   +1   → 补码:  -8
```

**只有最高位（MSB）的权重不同**：无符号是 +8，补码是 -8。
其余位完全一样，这是两种编码唯一的差异。

---

## 数值范围（以 4 位为例）

```
无符号:  0000(0) ~ 1111(15)     共 16 个值，从 0 开始
补码:    1000(-8) ~ 0111(7)     共 16 个值，不对称！
                  ↑
              |TMin| = 8 > TMax = 7
```

不对称的后果：`-TMin` 会溢出，所以有 `INT_MIN` 写法的问题（见下文）。

---

## 有符号 ↔ 无符号 转换

**位不变，解释方式变**。用 4 位演示：

```
位模式    无符号解释    补码解释
 1111        15          -1
 1110        14          -2
 1100        12          -4
 1000         8          -8
 0111         7           7   ← 分界线（TMax）
 0001         1           1
 0000         0           0
```

规律：
- `0xxx` 开头 → 两种解释结果相同
- `1xxx` 开头 → 无符号值 = 补码值 + 16（即 + 2^4）

---

## C 语言的隐式转换陷阱

**混合运算时，有符号转为无符号**（不是转为更大的类型）：

```c
-1 < 0U     // -1 被转成无符号 → UINT_MAX(4294967295) < 0？→ false ！
 2 > -1U    // -1U = UINT_MAX，2 < UINT_MAX → false ！
```

转换方向记忆：**有符号是"弱势"的一方，会被转换**。

**常量需要显式声明 U 后缀**：

```c
unsigned a = 3000000000;    // 警告：字面量默认有符号，3e9 超 INT_MAX
unsigned b = 3000000000U;   // 正确
```

**INT_MIN 为什么不能直接写 -2147483648**：

```c
// 错误写法
int x = -2147483648;
// 编译器先看到字面量 2147483648，它超过了 INT_MAX(2147483647)
// 编译器把它升级为 long，取负号后类型变成 long，赋给 int 是截断

// 正确写法
#define INT_MIN  (-INT_MAX - 1)   // 先用合法的 INT_MAX，再减 1
```

## 扩展与截断

**扩展（小类型 → 大类型）**：

```c
// 无符号：高位补 0
uint8_t  a = 0b00001010;   //  10
uint16_t b = a;            //  0000 0000 0000 1010  →  10  ✓

// 有符号（补码）：高位复制符号位
int8_t   c = 0b10001010;   // -118
int16_t  d = c;            //  1111 1111 1000 1010  → -118  ✓
```

**截断（大类型 → 小类型）**：丢掉高位，数值可能改变

```c
int x = 0x12345678;
char c = x;   // 只保留最低 8 位 0x78 = 120，高位全丢
```

---

## 无符号导致的三个经典 Bug

**Bug 1：循环变量是无符号，条件永远为真**

```c
for (unsigned i = n-1; i >= 0; i--)  // unsigned 永远 >= 0，死循环
    arr[i];
```

**Bug 2：strlen 相减绕回**

```c
// strlen 返回 size_t（无符号）
if (strlen(s) - strlen(t) > 0)   // 若 t 比 s 长：0 - 5 = UINT_MAX，> 0 为 true！
    puts("s is longer");
```

**Bug 3：length=0 时上界绕回**

```c
for (unsigned i = 0; i <= length - 1; i++)
//  length = 0 时：0 - 1 = UINT_MAX，循环跑 40 亿次
```

---

## 工程关联

- C++ 容器 `.size()` 返回 `size_t`，与 `int` 比较会触发隐式转换警告，正确写法是强转：`(int)vec.size()`
- Linux 内核大量用 `u32/u64`，代码审计时须注意无符号绕回（wraparound）
- TCP 序列号用无符号绕回实现循环计数，这是有意为之的设计

## 实验题：有符号/无符号隐式转换陷阱

**文件**：`Chapter2/2.2/exp_implicit_conv.cpp`
**编译**：`g++ -std=c++17 -Wall -Wconversion -o exp_implicit_conv exp_implicit_conv.cpp`

### 任务

写一个 C++ 程序，包含以下四个函数，每个函数演示一种转换陷阱，**打印出转换前后的实际数值**，让结果可观察。

**函数 1**：`void exp_mixed_compare()`

- 定义 `int a = -1` 和 `unsigned int b = 0`
- 打印 `a < b` 的结果（会是 `false`，为什么？）
- 打印 `(unsigned int)a` 的十六进制值，解释发生了什么

**函数 2**：`void exp_strlen_sub()`

- 定义两个字符串，令 `strlen(s) < strlen(t)`
- 打印 `strlen(s) - strlen(t)` 的值（注意 `strlen` 返回 `size_t`）
- 打印 `strlen(s) - strlen(t) > 0` 的结果，应为 `true`（违反直觉！）

**函数 3**：`void exp_unsigned_loop()`

- 用 `std::vector<int> v = {10, 20, 30}` 做倒序遍历
- 用 `size_t i = v.size() - 1; i >= 0; i--` 这个**错误写法**，观察 `i` 在 `0` 之后变成什么值
- 加 guard 限制只跑 5 次，打印每次的 `i` 值

**函数 4**：`void exp_sizet_int()`

- 定义 `std::vector<int> v(3)` 和 `int n = -1`
- 打印 `n < (int)v.size()` 和 `n < v.size()` 的结果对比
- 打印 `(size_t)n` 的值，解释差异来源
