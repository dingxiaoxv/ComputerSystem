# §3.4 访问信息 总结

## 核心主线

这一节的重点不是“背指令”，而是建立一个稳定心智模型：

- **寄存器**是 CPU 内部的超快临时存储
- **操作数**可以来自立即数、寄存器、内存
- **`mov`/`push`/`pop`** 是最基础的数据流动方式
- 读懂一条汇编，先问自己：**数据从哪来，要到哪去，宽度是多少**

---

## 1. 寄存器：同一块硬件的不同视图

x86-64 有 16 个 64 位通用寄存器。最重要的不是把名字全背下来，而是理解：

- 一个寄存器可以按不同宽度访问
- 小宽度写入时，行为不完全一样

| 64 位 | 32 位 | 16 位 | 8 位 | 常见用途 |
|------|------|------|------|----------|
| `%rax` | `%eax` | `%ax` | `%al` | 返回值、临时值 |
| `%rbx` | `%ebx` | `%bx` | `%bl` | 常作被保存寄存器 |
| `%rcx` | `%ecx` | `%cx` | `%cl` | 计数、移位计数 |
| `%rdx` | `%edx` | `%dx` | `%dl` | 参数、临时值 |
| `%rsi` | `%esi` | `%si` | `%sil` | 参数 |
| `%rdi` | `%edi` | `%di` | `%dil` | 参数 |
| `%rsp` | `%esp` | `%sp` | `%spl` | 栈顶指针 |
| `%rbp` | `%ebp` | `%bp` | `%bpl` | 栈帧基址（传统上） |

### 最重要的一条规则

- 写 32 位寄存器（如 `%eax`）时，**高 32 位会被自动清零**
- 写 8 位或 16 位寄存器时，**其余高位不会自动清零**

例子：

```asm
movq $-1, %rax        # rax = 0xffffffffffffffff
movl $1, %eax         # rax = 0x0000000000000001
movw $2, %ax          # rax = 0x0000000000000002
```

第一步后 `%rax` 全是 1；第二步因为写的是 `%eax`，高 32 位被清零；第三步只改低 16 位。

---

## 2. 操作数：立即数、寄存器、内存

大多数指令的源操作数和目的操作数都落在这三类里：

| 类型 | 例子 | 含义 |
|------|------|------|
| 立即数 | `$8`、`$-16` | 常量，直接写在指令里 |
| 寄存器 | `%rax`、`%rdi` | 从寄存器读/写 |
| 内存 | `(%rax)`、`8(%rbp)` | 访问某个地址对应的内存 |

### 内存寻址的统一写法

AT&T 语法常写成：

```asm
Imm(rb, ri, s)
```

表示有效地址：

```text
地址 = Imm + R[rb] + R[ri] * s
```

其中：

- `Imm` 是常量偏移
- `rb` 是基址寄存器
- `ri` 是变址寄存器
- `s` 是比例因子，只能是 `1/2/4/8`

例子：

| 表达式 | 含义 |
|--------|------|
| `(%rax)` | 访问 `rax` 指向的内存 |
| `8(%rbp)` | 访问 `rbp + 8` 地址处的内存 |
| `(%rax,%rdx,4)` | 访问 `rax + rdx*4` |
| `16(%rax,%rdx,8)` | 访问 `rax + rdx*8 + 16` |

这正是数组和结构体字段访问在汇编里的底层形式。

### 两个高频限制

- **普通 `mov` 不能内存到内存直接搬运**
- `%rsp` **不能作为变址寄存器**

所以这类代码必须借助寄存器中转：

```asm
movq (%rdi), %rax
movq %rax, (%rsi)
```

## 3. 数据传送：`mov` 是最基础的“数据流”

### 基本形式

| 指令 | 宽度 |
|------|------|
| `movb` | 1 字节 |
| `movw` | 2 字节 |
| `movl` | 4 字节 |
| `movq` | 8 字节 |

你读一条 `mov`，核心只看三件事：

1. 源操作数是谁
2. 目的操作数是谁
3. 这次搬多少字节

### 扩展传送

当源更窄、目标更宽时，需要明确怎么补高位：

- **符号扩展**：保留正负号
- **零扩展**：高位补 0

常见场景：

```asm
movsbq %al, %rax      # 把 8 位有符号数扩展到 64 位
movzbl %al, %eax      # 把 8 位无符号数扩展到 32 位
cltq                  # 把 eax 符号扩展到 rax
```

### C 类型转换和扩展的对应关系

| C 转换 | 汇编直觉 |
|--------|----------|
| `char -> int` | 通常是符号扩展 |
| `unsigned char -> int` | 通常是零扩展 |
| `int -> long` | 常常需要扩展到 64 位 |
| `unsigned -> unsigned long` | 高位补 0 |

如果你搞不清一条扩展指令在干什么，就回到 C 的问题：**源类型是有符号还是无符号？**

## 4. `pushq` / `popq`：栈顶上的特殊数据传送

栈在 x86-64 上默认**向低地址增长**，`%rsp` 永远指向当前栈顶。

### `pushq src`

等价于：

```asm
subq $8, %rsp
movq src, (%rsp)
```

### `popq dest`

等价于：

```asm
movq (%rsp), dest
addq $8, %rsp
```

所以：

- `push` 先减 `%rsp`，再写内存
- `pop` 先读内存，再加 `%rsp`

这就是函数调用中保存寄存器、保存返回地址、构造栈帧的基础动作。

---

## 5. 这一节最容易错的点

| 易错点 | 正确认识 |
|--------|----------|
| 写 `%eax` 只改低 32 位 | 错。写 `%eax` 会把 `%rax` 高 32 位清零 |
| `movq (%rdi), (%rsi)` 可以直接搬内存 | 错。普通 `mov` 不能内存到内存 |
| `8(%rbp)` 里的 `8` 是第 8 个字节 | 不一定。它表示地址偏移，不是“第几个变量” |
| `pushq` 只是“把值放栈里” | 不完整。它还会修改 `%rsp` |
| 扩展指令只是“补位” | 不完整。关键是补的是符号位还是 0 |

---

## 6. 工程关联

- 看反汇编时，数组/结构体访问本质上都在看**有效地址怎么算**
- 理解 32 位写寄存器自动清零，能解释很多编译器为什么偏爱 `movl`
- 栈上的保存/恢复动作，本质就是 `push`/`pop` 或等价的 `sub`/`mov`/`add`
- 调试 crash、看 core dump、分析 ABI 问题时，经常要顺着寄存器值和内存地址追数据流

## 7. 实验题 / 思考题

### 题 1：观察“写 `%eax` 会清零高 32 位”

写一个最小函数：

```c
unsigned long f(unsigned long x) {
    unsigned y = (unsigned)x;
    return y;
}
```

要求：

- 用 `gcc -Og -S` 或 `objdump -d` 看汇编
- 解释为什么编译器常用 32 位寄存器完成这个转换
- 回答：返回前 `%rax` 的高 32 位为什么是 0

### 题 2：比较有符号扩展和零扩展

写两个函数：

```c
long f1(char x) { return x; }
long f2(unsigned char x) { return x; }
```

要求：

- 观察两者汇编差异
- 说明哪个会保留符号位，哪个会高位补 0
- 试着输入 `0xff`，解释两者返回值为什么不同

### 题 3：证明“普通 `mov` 不能内存到内存”

写一个交换函数：

```c
void copy(long *src, long *dst) {
    *dst = *src;
}
```

要求：

- 反汇编后找出中间寄存器
- 解释为什么编译器必须先把 `*src` 读到寄存器，再写到 `*dst`
- 顺手标出源操作数和目的操作数

### 题 4：手算几个有效地址

假设：

- `%rax = 0x1000`
- `%rdx = 3`

计算下面几个地址：

1. `8(%rax)`
2. `(%rax,%rdx,4)`
3. `16(%rax,%rdx,8)`

然后回答：

- 哪一个最像 `a[i]`
- 哪一个最像“结构体字段 + 数组下标”混合访问

### 题 5：验证 `pushq` / `popq` 对 `%rsp` 的影响

写一个简单函数，编译后观察函数开头和结尾：

```c
long g(long x) {
    long y = x + 1;
    return y;
}
```

要求：

- 看编译器是否生成 `pushq`/`popq`
- 如果没有，再试 `-Og`、`-fno-omit-frame-pointer`
- 解释每次 `pushq` 和 `popq` 后 `%rsp` 分别如何变化

### 题 6：反推 C 语义

给你一段汇编：

```asm
movq 8(%rdi), %rax
movq %rax, (%rsi)
```

回答：

1. `%rdi` 和 `%rsi` 大概率分别是什么类型
2. 这两条指令对应的 C 语句可能是什么
3. 第一条为什么不是把“地址 8”读到 `%rax`

---

## 三句话总结

1. x86-64 的寄存器不是一堆分散名字，而是同一块寄存器文件的不同宽度视图
2. 读汇编时最重要的是盯住“数据从哪来、到哪去、宽度是多少、地址怎么算”
3. `mov`、扩展传送、`push/pop` 构成了后续理解函数调用、栈帧和地址计算的基础
