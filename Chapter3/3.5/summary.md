# §3.5 算术和逻辑操作 总结

## 核心主线

这一节的主线是：**同一块 ALU，四类操作（地址/一元/二元/移位）各自有什么规律，哪些地方会踩坑**

- `leaq` 不访问内存，是编译器的"免费算术"
- 一元/二元指令是最直接的 ALU 操作
- 移位指令的移位计数有严格限制：只能是立即数或 `%cl`
- 特殊算术（128 位乘法、有符号/无符号除法）有专用指令和隐式操作数

---

## 1. `leaq`：加载有效地址

### 本质

`leaq src, dst` 的作用是：**把地址算出来，存到 dst，完全不访问内存**。

```asm
leaq 8(%rdi,%rsi,4), %rax   # rax = rdi + rsi*4 + 8，没有任何内存读写
```

### 和 `movq` 的区别

```asm
movq 8(%rdi), %rax   # 访问内存：把地址 (rdi+8) 处的值读到 rax
leaq 8(%rdi), %rax   # 不访问内存：把 rdi+8 这个数本身存到 rax
```

### 编译器用 `leaq` 做快速算术

这是 `leaq` 最常被误认的用法。编译器经常用它完成乘以小常数的运算：

```c
long f(long x) { return x * 5; }
```

生成的汇编可能是：

```asm
leaq (%rdi,%rdi,4), %rax   # rax = rdi + rdi*4 = 5*rdi
```

常见 `leaq` 算术模式：

| `leaq` 写法 | 等价 C 表达式 |
|------------|--------------|
| `leaq (%rax,%rax,1), %rax` | `x * 2` |
| `leaq (%rax,%rax,2), %rax` | `x * 3` |
| `leaq (%rax,%rax,4), %rax` | `x * 5` |
| `leaq (%rax,%rax,8), %rax` | `x * 9` |
| `leaq 7(%rax), %rax` | `x + 7` |

多个 `leaq` 可以组合：

```asm
leaq (%rdi,%rdi,2), %rax   # rax = 3x
leaq (%rax,%rax,4), %rax   # rax = 5 * 3x = 15x
```

### 关键规则

- 比例因子 `s` 只能是 `1/2/4/8`，所以 `leaq` 无法直接乘以 6、7、10 等
- 不设置条件码（EFLAGS）——这点和算术指令不同

## 2. 一元操作和二元操作

### 一元操作（只有一个操作数）

| 指令 | 操作 |
|------|------|
| `inc D` | D += 1 |
| `dec D` | D -= 1 |
| `neg D` | D = -D（取补码相反数） |
| `not D` | D = ~D（按位取反） |

操作数可以是寄存器或内存地址。

### 二元操作（两个操作数）

| 指令 | 操作 |
|------|------|
| `add S, D` | D = D + S |
| `sub S, D` | D = D - S |
| `imul S, D` | D = D \* S（截断到 64 位） |
| `xor S, D` | D = D ^ S |
| `or S, D` | D = D \| S |
| `and S, D` | D = D & S |

### 最容易搞混的点

**AT&T 语法：源操作数在左，目的操作数在右。**

```asm
subq %rax, %rdx   # rdx = rdx - rax，不是 rax - rdx
```

第一次读汇编的人很容易以为是"rax 减去 rdx"——记住"目的在右"。

### `xor` 的零化用途

```asm
xorl %eax, %eax   # eax = 0（最短的清零方式，比 movl $0, %eax 更短）
```

编译器常用 `xorl %eax, %eax` 代替 `movl $0, %eax`，因为它能用 4 字节编码而前者需要 5 字节（写 `%eax` 会自动清零 `%rax` 高 32 位）。

## 3. 移位操作

### 四条指令

| 指令 | 操作 |
|------|------|
| `sal` / `shl` | 左移（两者等价，都是逻辑左移） |
| `sar` | 算术右移（高位补符号位） |
| `shr` | 逻辑右移（高位补 0） |

### 移位量的限制

**移位量只能是立即数或 `%cl` 寄存器（只用低 6 位，因为最大有效移位是 63）。**

```asm
salq $3, %rax      # 立即数移位：rax <<= 3
salq %cl, %rax     # 寄存器移位：rax <<= cl（只用低 6 位）
salq %rax, %rdx    # 错误：不能用任意寄存器作移位量
```

### 算术右移 vs 逻辑右移

```c
int x = -8;
x >> 1;             // sar：高位补 1，结果 -4（算术右移）
(unsigned)x >> 1;   // shr：高位补 0，结果很大的正整数（逻辑右移）
```

这对应 §2.3 学过的"有符号数右移需要算术右移"。

### 移位替代除法的编译器惯用法

```c
long div4(long x) { return x / 4; }
```

生成的汇编：

```asm
sarq $2, %rdi       # rdi >>= 2（但负数有问题）
```

对负数，除法结果应向 0 取整而非向下取整，所以编译器实际会先加偏置：

```asm
movq %rdi, %rax
sarq $63, %rax      # rax = x >> 63，全 1（负数）或全 0（正数）
andl $3, %eax       # eax = 0 或 3（偏置量 = 2^k - 1）
addq %rax, %rdi     # x += 偏置
sarq $2, %rdi       # 再右移
```

这正是 §2.3.7 讲的"除以 2 的幂需要加偏置"在汇编层面的体现。

## 4. 特殊算术操作（§3.5.5）

### 128 位乘法

64 位 × 64 位 = 128 位，无法用单个寄存器存放。x86-64 用 `%rdx:%rax` 这对寄存器存 128 位结果：

| 指令 | 操作 |
|------|------|
| `imulq S` | 有符号 64×64→128：`%rdx:%rax = %rax * S` |
| `mulq S` | 无符号 64×64→128：`%rdx:%rax = %rax * S` |

注意 `imulq` 有两种形式：
- **双操作数**：`imulq S, D`（D = D * S，截断到 64 位，有符号/无符号等价）
- **单操作数**：`imulq S`（128 位结果，隐式用 `%rax` 作被乘数）

### 128 位除法

| 指令 | 操作 |
|------|------|
| `cqto` | 把 `%rax` 符号扩展到 `%rdx:%rax`（为 `idivq` 准备） |
| `idivq S` | 有符号除法：商→`%rax`，余数→`%rdx` |
| `divq S` | 无符号除法：商→`%rax`，余数→`%rdx` |

典型使用模式：

```asm
movq x, %rax       # 被除数低 64 位
cqto               # 符号扩展到 rdx:rax
idivq y            # 除以 y：商在 rax，余数在 rdx
```

### 为什么这些是"特殊"的

普通 `imul S, D` 丢弃高 64 位，适合一般乘法。只有需要完整 128 位结果（如高精度运算）或 `%` 取余操作才会用到单操作数 `imulq/idivq`。

## 5. 这一节最容易错的点

| 易错点 | 正确认识 |
|--------|----------|
| `leaq` 读了内存 | 错。`leaq` 只做地址计算，不访问内存 |
| `leaq` 只用于取地址 | 错。编译器大量用它做快速整数算术（乘以小常数） |
| `subq %rax, %rdx` 是 rax 减 rdx | 错。AT&T 语法目的在右：rdx = rdx - rax |
| 移位量可以是任意寄存器 | 错。只能是立即数或 `%cl`（唯一能做移位量的寄存器） |
| `sar` 和 `shr` 等价 | 错。`sar` 补符号位，`shr` 补 0，对负数结果截然不同 |
| `imulq` 只有一种形式 | 错。双操作数截断到 64 位，单操作数产生 128 位结果 |

---

## 6. 工程关联

- 性能分析时看到 `leaq` 密集，说明编译器在做快速算术，不是在访问内存
- `xorl %eax, %eax` 是生成最短清零代码的惯用法，`-O2` 汇编里很常见
- 负数整数除法的编译产物里一定有加偏置的 `sar + and + add` 组合，不要误读为独立操作
- `%cl` 作移位量的限制来自 x86 设计，在处理动态移位时不能随意用其他寄存器

## 7. 实验题

### 题 1：`leaq` 作快速乘法

写三个函数：

```cpp
long mul5(long x) { return x * 5; }
long mul15(long x) { return x * 15; }
long mul6(long x) { return x * 6; }
```

要求：

- `gcc -O1 -S` 生成汇编
- 找出哪些用了 `leaq`，哪些没用（`mul6` 因为 6 不是 `1+2^k`，看编译器怎么处理）
- 手算 `mul15` 用两个 `leaq` 是如何组合的

### 题 2：观察算术右移 vs 逻辑右移

写两个函数：

```cpp
long signed_shift(long x) { return x >> 3; }
unsigned long unsigned_shift(unsigned long x) { return x >> 3; }
```

要求：

- 查看汇编，确认一个用 `sarq`，另一个用 `shrq`
- 在 C++ 程序里分别输入 `-8` 和 `0xffffffffffffffff8`，验证结果
- 解释为什么负数算术右移结果是 `-1`（全 1）而不是 0

### 题 3：反推 `sub` 的操作数顺序

给出下面汇编片段，不允许查文档，通过写 C 代码对比验证来确认语义：

```asm
movq %rdi, %rax
subq %rsi, %rax
```

要求：

- 先用直觉猜：这是 `rdi - rsi` 还是 `rsi - rdi`？
- 写两个函数 `long f1(long a, long b) { return a-b; }` 和 `return b-a`，对比汇编
- 给出 AT&T 语法操作数顺序的结论

### 题 4：观察负数除法的偏置修正

写一个函数：

```cpp
long div_by_4(long x) { return x / 4; }
```

要求：

- `-O1` 编译查看汇编
- 找出加偏置的 `sar`/`and`/`add` 序列
- 手动验证：当 `x = -7` 时，加偏置后移位的结果应该是 `-1`（C 的截断除法）而非 `-2`（向下取整）

### 题 5：128 位乘法观察

写一个函数并用 `__int128` 观察行为：

```cpp
// 先用这个触发单操作数 imulq
__int128 wide_mul(long a, long b) {
    return (__int128)a * b;
}

// 再对比普通 64 位乘法
long normal_mul(long a, long b) {
    return a * b;
}
```

要求：

- 查看汇编，找到单操作数 `imulq` 和 `%rdx:%rax` 的使用
- 对比双操作数 `imulq` 的汇编
- 说明：对于溢出场景，`__int128` 保留了哪些 64 位版本丢失的信息

---

## 三句话总结

1. `leaq` 的核心不是"取地址"而是"地址计算单元可以免费做整数算术"，这是编译器优化乘以小常数的标准手段
2. AT&T 语法目的在右，`sub S, D` 是 `D -= S`；移位量必须是 `%cl`，这两个规则背不熟就会频繁误读汇编
3. 特殊算术（单操作数 `imulq/idivq`）的隐式操作数 `%rdx:%rax` 是 x86 遗留设计，出现时要立刻联想"128 位结果"或"求余数"
